## 团队成员贡献分分配规则

规则概述:

按任务分配一个初始的分数,每周会上公布一下一周的进度，然后讨论得出每个人的完成度，实现给分;
每天汇报当前项目，进度剩余时间，github上传每日的成果，测试人员检测成果并在GitHub上反馈测试结果

| 评分细则                   | 分数              | 补充说明               |
| -------------------------- | ----------------- | ---------------------- |
| 基础分                     | 40                | 固定基础值             |
| 完成个人任务               | 对任务分数之和    | 不同任务分数不同       |
| 对项目做出建设性促进       | +1                | 通过提出改进并做出优化 |
| 个人原因对项目进展造成阻碍 | -1                | 个人任务未完成等等     |
| 最终得分                   | 40+(得分/总分)*60 | 非整数分和重复分再细调 |
